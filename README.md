Pré-requisitos 
Antes de começar, certifique-se de ter o seguinte: 
•Uma conta no GitHub. Se você não tiver uma, pode criar uma gratuitamente em github.com. 
•O GitHub Desktop instalado em seu computador. Você pode baixá-lo em desktop.github.com. 
•Os arquivos do seu projeto prontos para serem adicionados ao repositório. 
 
Passo a Passo 
Passo 1: Criar um Novo Repositório no GitHub 
1. Acesse o GitHub: Abra seu navegador e vá para github.com. Faça login na 
sua conta. 
 

![Imagem](images/page2_img1.png)


## Página 3

 
2. Inicie a Criação do Repositório: No canto superior direito da página, clique 
no sinal de + e selecione New repository (Novo repositório). 
 
3. Configure o Repositório 
• Owner (Proprietário): Seu nome de usuário do GitHub será 
preenchido automaticamente. 
• Repository name (Nome do repositório): Digite um nome para o seu 
projeto (ex: GaleriaDeArteJPA). 
• Description (Descrição - opcional): Adicione uma breve descrição 
do seu projeto (ex: Tafera IFMT materia de Persistência Orientada a 
Objeto).

## Página 4

• Public/Private: Escolha se o repositório será público (visível para 
todos) ou privado (visível apenas para você e quem você convidar). 
• Initialize this repository with: Marque a opção Add a README file 
(Adicionar um arquivo README). Isso é uma boa prática para

Passo 2: Clonar o Repositório para o Seu Computador Local 
Após criar o repositório no GitHub, você precisará cloná-lo para o seu 
computador para poder adicionar os arquivos do seu projeto.

## Página 5

1. Copie a URL do Repositório: Na página do seu novo repositório no GitHub, 
clique no botão Code (Código) e, em seguida, copie a URL HTTPS. 
 
 
Abra o GitHub Desktop: Inicie o aplicativo GitHub Desktop no seu computador. 
Clone o Repositório: 
•Crie uma pasta para clonar os arquivos do git no PC. 
•No GitHub Desktop, vá em File (Arquivo) > Clone repository (Clonar 
repositório). 
•Na janela que se abre, selecione a aba URL. 
•Cole a URL que você copiou do GitHub no campo Repository URL.

## Página 12

 
Passo 4: Fazer Commit e Push das Alterações 
• Com os arquivos adicionados, você precisa registrar essas 
alterações (commit) e enviá-las para o GitHub (push). 
• Verifique as Alterações no GitHub Desktop: Volte para o GitHub 
Desktop. Você verá uma lista das alterações (arquivos adicionados, 
modificados ou excluídos) na aba Changes (Alterações). 
• Escreva a Mensagem de Commit: No campo Summary (Resumo), 
digite uma breve descrição das alterações (ex: Adicionando os 
arquivos do projeto). Você pode adicionar uma descrição mais 
detalhada no campo Description (Descrição), se desejar. 
• Faça o Commit: Clique no botão Commit to main (Fazer commit 
para main). 
• Faça o Push: Após o commit, um botão Push origin (Enviar para 
origin) aparecerá. Clique nele para enviar suas alterações para o 
repositório remoto no GitHub.

## Página 15

 
 
 
Passo 6: Abrir o Projeto no NetBeans 
Usando a IDE NetBeans, pode abrir o projeto diretamente da pasta clonada. 
• Abra o NetBeans: Inicie o NetBeans IDE. 
• Abra o Projeto: Vá em File (Arquivo) > Open Project (Abrir Projeto). 
• Navegue até a Pasta: Navegue até a pasta local do seu repositório 
(onde você clonou o projeto no Passo 2.3) e selecione a pasta raiz do 
seu projeto. 
• Abra: Clique em Open Project (Abrir Projeto).
